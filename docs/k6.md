# k6 load tests (production smoke)

[k6](https://k6.io/) scripts under `tests/k6/` validate waitlist security on a deployed site.

## Install k6

```bash
# macOS
brew install k6

# Or: https://grafana.com/docs/k6/latest/set-up/install-k6/
```

## Run against production

```bash
export BASE_URL=https://obsessions-snowy.vercel.app

# 1) Login page: waitlist copy + API rejects missing captcha (Turnstile on server)
k6 run tests/k6/waitlist-smoke.js

# 2) Rate limit: 10 concurrent POSTs → expect >= 5× HTTP 429
k6 run tests/k6/waitlist-rate-limit.js
```

### Rate-limit test behavior

- All 10 VUs share the runner’s public IP (same bucket as a single user spamming).
- The API checks **rate limit before Turnstile**, so missing captcha tokens still consume the limit.
- Requests 6–10 should return `429` with `"Too many requests"` when `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` are set on Vercel.

If every request returns `400` Captcha and zero `429`, Upstash is likely not configured or the deploy is stale.

### Smoke test: Turnstile check failed

The smoke step does **not** scrape HTML for the widget (Next.js loads it in the browser). It POSTs without a token and expects **400** + `"Captcha verification failed"`.

If that check fails:

- Add `TURNSTILE_SECRET_KEY` and `NEXT_PUBLIC_TURNSTILE_SITE_KEY` on **Vercel**, then **redeploy**
- Confirm the widget appears on `/login` → Waitlist in a real browser

### Successful submit (manual, optional)

Turnstile tokens are short-lived. To test a full `200` join:

1. Open `/login` → Waitlist, complete the widget.
2. Copy `turnstileToken` from the browser network tab.
3. Run:

```bash
K6_TURNSTILE_TOKEN='paste-token-here' k6 run tests/k6/waitlist-submit.js
```

## Local / preview

```bash
BASE_URL=http://localhost:3000 k6 run tests/k6/waitlist-smoke.js
```

Rate-limit test only applies when Upstash env vars are set in that environment.

## CI

Run manually from GitHub: **Actions → k6 waitlist smoke → Run workflow**.

Optional inputs:

- **site_url** — default `https://obsessions-snowy.vercel.app`
- **run_rate_limit** — disable to only run login + captcha API smoke (skips 10× POST spam test)

The rate-limit job uses the GitHub runner IP. Re-running within an hour may hit 429 on the smoke POSTs; wait or disable **run_rate_limit** for a quick UI check.

k6 is not run on every PR (requires Upstash on the target URL for the rate-limit test to pass).
